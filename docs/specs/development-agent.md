# 开发代理能力

## 背景

用户需要一个能自主执行开发任务的代理，从模糊需求到代码提交全链路自动化。

## 需求

### 1.1 任务拆解与委派

接收用户的模糊目标（一句话需求），拆解为具体、可执行的开发任务，委派给编码 Agent（如 Cursor）执行。

- 将"加一个用户注册功能"拆为数据库建表、API 设计、前端页面等子任务
- 每个子任务有明确的验收标准
- 按依赖关系排序执行
- 单次委派聚焦一个明确任务，复杂目标拆成多次

### 1.2 代码质量验证

编码 Agent 完成后，检查产出是否符合预期。

- 审查变更文件列表和 diff
- 运行测试套件，检查是否通过
- 检查构建是否成功
- 发现问题时决定是重试、修正指令还是上报用户

### 1.3 版本控制

管理代码的版本历史。

- 提交变更（commit），附有清晰的描述
- 推送到远端仓库
- 查看变更历史和差异
- 处理分支、合并冲突

### 1.4 技术调研

在需要时搜索互联网，查阅文档和社区经验。

- 遇到报错时搜索解决方案
- 查阅 API 文档和库的用法
- 了解技术选型的社区评价
- 替用户做技术方案对比

### 1.5 错误诊断与恢复

分析错误日志和异常，定位根因并尝试修复。

- 阅读 stack trace，定位出错的代码位置
- 区分环境问题、代码 Bug、配置错误
- 自动尝试修复或给出修复建议
- 记录踩坑经验，避免重复犯错

### 2.1 任务管理

维护一个任务队列，跟踪每个任务的生命周期。

- 创建任务，设置优先级（紧急 / 重要 / 一般）
- 跟踪状态：排队中 → 执行中 → 已完成 / 失败
- 管理任务间依赖关系
- 支持多任务并行执行，监控整体进度

### 2.2 进度汇报

主动向用户汇报工作进展，格式结构化、重点突出。

- 做了什么 → 结果如何 → 需要用户做什么
- 只在需要时展开技术细节

### 2.3 决策记录与文档维护

记录技术决策，保持文档同步。

## 验收标准

- [ ] 用户下达模糊目标 → Agent 拆解为可执行任务并逐个完成
- [ ] Cursor CLI 调用有完整审计日志
- [ ] 代码变更可通过 git diff 追溯
- [ ] 任务完成后自动更新 STATUS.md

## 技术约束

- 代码修改只通过 Cursor Agent 执行（`bash pty:true` + `cursor agent -p`）
- 希露菲本体不直接修改代码文件
