# 产品定义

希露菲是用户的**全权代理数字分身**——不是被动的聊天助手，而是有自主决策权的贴身侍从。用户下达目标（一句话甚至一个语音），希露菲自行拆解任务、调度工具、执行操作、汇报结果。

## 用户画像

独立开发者/研究者，同时管理多个项目，需要一个"另一个自己"来处理日常事务：写代码、查邮件、看论文、管家居、记人情。碎片化场景多（通勤、做饭、散步），语音是主要输入方式。

## 交互渠道

| 渠道 | 定位 | 输入 | 输出 |
|------|------|------|------|
| **风铃** (Web :3001) | 桌面日常 | 语音（按住说话）/ 文字 / 图片 / 视频 | 语音问→文字+语音朗读；文字问→纯文字 |
| **Telegram Bot** | 随时随地 | 文字 / 语音 / 图片 / 视频 | 语音问→文字+语音消息；文字问→纯文字 |
| **OpenCami** (Web :3000) | 复杂任务 | 文字 + 工具调用面板 | 文字 + 工具执行过程展示 |

三个渠道共享同一个 Agent 后端（OpenClaw Gateway），人设、记忆、工具链完全一致。

## 交互原则

### 条件语音

输入方式自动决定回复形态：

- **语音输入** → 文字 + 逐句流式语音朗读（用户在移动/开车，需要听）
- **文字输入** → 纯文字（用户在看屏幕，不需要声音）

### 内容分区

AI 回复自动区分"对话文本"和"附件"：

- **文本**（日常对话）→ 显示 + 朗读
- **附件**（代码、命令、配置）→ 只显示不朗读，带语法高亮和复制按钮

### 即时响应

任何需要等待的操作，先确认用户再异步执行：

- "10 分钟后提醒我开会" → 立即回复"已设好" → 10 分钟后主动通知
- "帮我重构 auth 模块" → 立即回复"已开始，预计 5 分钟" → 完成后汇报
- 用户不会被阻塞，随时可以继续聊其他事

### 主动推进

希露菲不被动等指令。发现异常主动告警，任务完成主动汇报，接近 deadline 主动提醒。

## 审计与安全

- 所有对话交互记录到 `data/voice-audit/` (JSONL, 按日存储)
- 请求到达立即记录（`req`），响应完成再记录（`res`）
- 全部本地运行，数据不外泄
- 破坏性操作前确认，外部通信先问
