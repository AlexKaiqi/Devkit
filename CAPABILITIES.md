# 数字分身能力清单

Kite 作为用户的数字分身，需要具备以下技能。
本文档只描述**需要什么能力**，不绑定任何具体产品或平台。

---

## 一、开发技能

### 1.1 任务拆解与委派

接收用户的模糊目标（一句话需求），拆解为具体、可执行的开发任务，
委派给编码 Agent（如 Cursor）执行。

- 将"加一个用户注册功能"拆为数据库建表、API 设计、前端页面等子任务
- 每个子任务有明确的验收标准
- 按依赖关系排序执行
- 单次委派聚焦一个明确任务，复杂目标拆成多次

### 1.2 代码质量验证

编码 Agent 完成后，检查产出是否符合预期。

- 审查变更文件列表和 diff
- 运行测试套件，检查是否通过
- 检查构建是否成功
- 发现问题时决定是重试、修正指令还是上报用户

### 1.3 版本控制

管理代码的版本历史。

- 提交变更（commit），附有清晰的描述
- 推送到远端仓库
- 查看变更历史和差异
- 处理分支、合并冲突

### 1.4 技术调研

在需要时搜索互联网，查阅文档和社区经验。

- 遇到报错时搜索解决方案
- 查阅 API 文档和库的用法
- 了解技术选型的社区评价
- 替用户做技术方案对比

### 1.5 错误诊断与恢复

分析错误日志和异常，定位根因并尝试修复。

- 阅读 stack trace，定位出错的代码位置
- 区分环境问题、代码 Bug、配置错误
- 自动尝试修复或给出修复建议
- 记录踩坑经验，避免重复犯错

---

## 二、项目管理技能

### 2.1 任务管理

维护一个任务队列，跟踪每个任务的生命周期。

- 创建任务，设置优先级（紧急 / 重要 / 一般）
- 跟踪状态：排队中 → 执行中 → 已完成 / 失败
- 管理任务间依赖关系（A 完成后再启动 B）
- 用户随时可查询"当前有哪些事在做"
- 支持多任务并行执行，监控整体进度

### 2.2 进度汇报

主动向用户汇报工作进展，格式结构化、重点突出。

- 做了什么 → 结果如何 → 需要用户做什么
- 只在需要时展开技术细节
- 多个任务时给出整体进度概览

### 2.3 决策记录

记录开发过程中做出的技术决策及原因。

- 技术选型的考量（为什么选 A 不选 B）
- 架构变更的原因
- 需要用户确认的决策单独标记
- 形成可追溯的决策历史

### 2.4 文档维护

保持项目文档与实际状态同步。

- 项目目标、当前状态、决策记录等核心文档
- 在代码结构发生重大变更时更新相关文档
- 文档是用户和 Agent 之间的信息桥梁

## 三、沟通技能

### 3.1 多渠道对话

通过文字和语音与用户实时沟通，支持多端。

- 桌面端：在电脑上查看对话和操作记录
- 移动端：在手机上随时发消息、下指令
- 消息历史跨设备同步
- 支持 Markdown（代码块、表格、列表）

### 3.2 语音交互

支持语音输入和语音输出，适合碎片化沟通。

- 语音输入：用户说话 → 转为文字 → Agent 理解并执行
- 语音输出：Agent 回复 → 转为语音 → 用户听取
- 中文识别准确，延迟低
- 适合手机端、通勤、开车等场景

### 3.3 主动通知

Agent 在关键事件发生时主动找用户，而非被动等待。

- 任务完成或失败时立即通知
- 发现异常时告警（测试失败、服务挂了、仓库有冲突）
- 接近 deadline 时提醒
- 避免打扰：根据时间和重要性决定是否通知

### 3.4 视觉理解

理解用户发送的图片（截图、设计稿、照片）。

- 用户拍屏幕上的报错截图，Agent 能看懂并分析
- 查看 UI 截图判断样式是否符合预期
- 理解设计稿中的布局、配色、交互要求

## 四、知识管理技能

### 4.1 经验积累

从每次开发中学习，把经验沉淀下来。

- 记录踩过的坑和解决方案
- 记录用户的偏好和习惯
- 记录项目的技术栈、架构特点
- 新经验不覆盖旧经验，形成增量知识库

### 4.2 跨会话记忆

每次对话结束后不失忆，下次能接上。

- 会话开始时自动加载最近的上下文
- 长期重要信息持久保存
- 支持按关键词检索历史记忆
- "上次说的那个问题"能回忆起来

### 4.3 知识检索

在积累的知识中快速找到相关信息。

- 按错误关键词搜索踩坑记录
- 按项目名搜索历史决策
- 按日期范围查看工作日志
- 语义搜索（按含义而非精确匹配）为佳

---

## 五、自动化与监控技能

### 5.1 服务管理

启动、停止、监控本地开发服务。

- 一键启动完整开发环境（Gateway、STT Proxy、Tunnel 等）
- 检测服务是否正常运行
- 服务异常时自动重启或通知用户
- 查看服务日志

### 5.2 定时巡检

定期自动检查项目健康度，无需用户触发。

- 编码 Agent 是否运行超时
- 仓库是否有未提交的变更
- 文档是否过期
- 待确认的决策项是否积压

### 5.3 日程与提醒

理解时间概念，支持基于时间的行为。

- 知道当前时间、用户时区、是否在工作时间
- "明天上午提醒我 review 那个 PR"
- "这个任务拖了三天了" —— 能感知到并提醒
- deadline 提醒

## 六、外部集成技能

### 6.1 代码托管平台

与 GitHub / GitLab 等平台交互。

- 创建和管理 Issue
- 查看 PR 状态、代码审查意见
- 触发和查看 CI/CD 结果
- 查看仓库动态（新的 Issue、PR、Release）

### 6.2 团队协作平台

与飞书、Slack 等平台集成。

- 接收来自多个渠道的消息
- 跨平台消息转发和同步
- 在群聊中只在有价值时发言

### 6.3 第三方 API

按需调用外部服务。

- 云服务 API（部署、监控、日志）
- SaaS 工具（设计工具、数据库管理等）
- 以插件化方式扩展，不硬编码
