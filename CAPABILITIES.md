# 数字分身能力清单

希露菲作为用户的数字分身，需要具备以下技能。
第一至九章描述**需要什么能力**（不绑定产品），第十章推荐**用什么工具实现**（开源+CLI 优先）。

---

## 一、开发技能

### 1.1 任务拆解与委派

接收用户的模糊目标（一句话需求），拆解为具体、可执行的开发任务，
委派给编码 Agent（如 Cursor）执行。

- 将"加一个用户注册功能"拆为数据库建表、API 设计、前端页面等子任务
- 每个子任务有明确的验收标准
- 按依赖关系排序执行
- 单次委派聚焦一个明确任务，复杂目标拆成多次

### 1.2 代码质量验证

编码 Agent 完成后，检查产出是否符合预期。

- 审查变更文件列表和 diff
- 运行测试套件，检查是否通过
- 检查构建是否成功
- 发现问题时决定是重试、修正指令还是上报用户

### 1.3 版本控制

管理代码的版本历史。

- 提交变更（commit），附有清晰的描述
- 推送到远端仓库
- 查看变更历史和差异
- 处理分支、合并冲突

### 1.4 技术调研

在需要时搜索互联网，查阅文档和社区经验。

- 遇到报错时搜索解决方案
- 查阅 API 文档和库的用法
- 了解技术选型的社区评价
- 替用户做技术方案对比

### 1.5 错误诊断与恢复

分析错误日志和异常，定位根因并尝试修复。

- 阅读 stack trace，定位出错的代码位置
- 区分环境问题、代码 Bug、配置错误
- 自动尝试修复或给出修复建议
- 记录踩坑经验，避免重复犯错

## 二、项目管理技能

### 2.1 任务管理

维护一个任务队列，跟踪每个任务的生命周期。

- 创建任务，设置优先级（紧急 / 重要 / 一般）
- 跟踪状态：排队中 → 执行中 → 已完成 / 失败
- 管理任务间依赖关系（A 完成后再启动 B）
- 用户随时可查询"当前有哪些事在做"
- 支持多任务并行执行，监控整体进度

### 2.2 进度汇报

主动向用户汇报工作进展，格式结构化、重点突出。

- 做了什么 → 结果如何 → 需要用户做什么
- 只在需要时展开技术细节
- 多个任务时给出整体进度概览

### 2.3 决策记录

记录开发过程中做出的技术决策及原因。

- 技术选型的考量（为什么选 A 不选 B）
- 架构变更的原因
- 需要用户确认的决策单独标记
- 形成可追溯的决策历史

### 2.4 文档维护

保持项目文档与实际状态同步。

- 项目目标、当前状态、决策记录等核心文档
- 在代码结构发生重大变更时更新相关文档
- 文档是用户和 Agent 之间的信息桥梁

## 三、沟通技能

### 3.1 多渠道对话

通过文字和语音与用户实时沟通，支持多端多渠道。

- **风铃**（Web）：按住说话 · 流式朗读 · 大模型音色可选
- **Telegram Bot**：文字 + 语音，手机随时下指令
- **OpenCami**（Web）：全功能聊天界面，支持工具调用和任务调度
- 消息历史各渠道独立维护
- 回复内容自动分区：对话文本（朗读）+ 附件（代码块，仅展示）

### 3.2 语音交互

支持语音输入和语音输出，适合碎片化沟通。

- **语音输入**：用户说话 → 豆包 BigModel ASR → 文字 → Agent 执行
- **语音输出**：Agent 回复 → 豆包语音合成大模型（25+ 大模型音色）→ 用户听取
- **流式 TTS**：逐句合成，边生成边播放，显著降低首句延迟
- **文本/附件分离**：代码和命令不朗读，只展示
- 中文识别准确，延迟低
- 适合手机端、通勤、开车等场景

### 3.3 主动通知

Agent 在关键事件发生时主动找用户，而非被动等待。

- 任务完成或失败时立即通知
- 发现异常时告警（测试失败、服务挂了、仓库有冲突）
- 接近 deadline 时提醒
- 避免打扰：根据时间和重要性决定是否通知

### 3.4 视觉理解

理解用户发送的图片和视频。

- 用户拍屏幕上的报错截图，Agent 能看懂并分析
- 查看 UI 截图判断样式是否符合预期
- 理解设计稿中的布局、配色、交互要求
- 观看操作录屏，理解用户遇到的问题
- 查看 demo 视频，给出改进建议

## 四、知识管理技能

### 4.1 经验积累

从每次开发中学习，把经验沉淀下来。

- 记录踩过的坑和解决方案
- 记录用户的偏好和习惯
- 记录项目的技术栈、架构特点
- 新经验不覆盖旧经验，形成增量知识库

### 4.2 跨会话记忆

每次对话结束后不失忆，下次能接上。

- 会话开始时自动加载最近的上下文
- 长期重要信息持久保存
- 支持按关键词检索历史记忆
- "上次说的那个问题"能回忆起来

### 4.3 知识检索

在积累的知识中快速找到相关信息。

- 按错误关键词搜索踩坑记录
- 按项目名搜索历史决策
- 按日期范围查看工作日志
- 语义搜索（按含义而非精确匹配）为佳

## 五、自动化与监控技能

### 5.1 服务管理

启动、停止、监控本地开发服务。

- 一键启动完整开发环境（Gateway、STT Proxy、Tunnel 等）
- 检测服务是否正常运行
- 服务异常时自动重启或通知用户
- 查看服务日志

### 5.2 定时巡检

定期自动检查项目健康度，无需用户触发。

- 编码 Agent 是否运行超时
- 仓库是否有未提交的变更
- 文档是否过期
- 待确认的决策项是否积压

### 5.3 日程与提醒

理解时间概念，支持基于时间的行为。

- 知道当前时间、用户时区、是否在工作时间
- "明天上午提醒我 review 那个 PR"
- "这个任务拖了三天了" —— 能感知到并提醒
- deadline 提醒

### 5.4 App / GUI 操作

在手机、电脑桌面、网页上自主完成操作：看屏幕→理解→执行→验证。
覆盖三个操作面：

#### 5.4a 手机操作（Android）

在用户手机（或模拟器）上操控 App。

- 获取当前屏幕的结构化 UI 树（每个元素的名称、类型、精确坐标）
- 结合截图让视觉模型理解页面内容和状态
- 自主决策下一步操作（点击、输入、滑动、按键）
- 操作后重新获取状态，验证是否达成目标
- 能处理弹窗、引导页等意外情况

**已验证的工作流：**

```
UIAutomator dump_hierarchy → 结构化元素列表（精确坐标）
  + 标注截图 → 视觉模型理解+决策
    → ADB 执行操作（click / type / swipe / press）
      → 再次获取状态验证
```

**测试结果（2/2 通过）：**
- "打开 Chrome 浏览器"：2 步完成（识别图标 → 点击）
- "在 Chrome 地址栏输入 github.com 并访问"：3 步完成（输入 URL → 点击建议 → 确认页面加载）

**升级路径：** 火山方舟提供 `Doubao-1.5-UI-TARS` API（字节跳动专用 GUI Agent 模型），
可替代通用视觉模型做更精确的 GUI grounding，AndroidWorld 基准 46.6 分（GPT-4o 为 34.5）。

#### 5.4b 桌面操作（macOS）

在用户电脑桌面上操控应用程序。

- 截取屏幕 / 指定窗口 / 菜单栏，精确到像素
- 生成结构化 UI Map（元素 ID、标签、坐标）
- 模拟交互：点击、输入、滚动、拖拽、快捷键、菜单导航
- 结合视觉模型理解屏幕内容，自然语言驱动操作
- 多屏幕、多桌面支持

**典型场景：**
- 操作没有 CLI / API 的桌面应用（如微信、备忘录、系统设置）
- 辅助用户完成重复性 GUI 操作（填表、截图、整理文件）
- 验证桌面应用的 UI 是否符合预期

#### 5.4c Web 操作

在浏览器中自动化操作网页。

- 打开 URL、导航、等待页面加载
- 填写表单、点击按钮、选择下拉菜单
- 提取页面内容（文本、表格、链接）
- 截取网页截图用于视觉验证
- 处理登录、多步流程、动态加载

**典型场景：**
- 操作没有 API 的 Web 服务（如内部系统、论坛、管理后台）
- 自动化表单提交、信息采集
- 前端开发中的 UI 自动化测试

## 六、外部集成技能

### 6.1 代码托管平台

与 GitHub / GitLab 等平台交互。

- 创建和管理 Issue
- 查看 PR 状态、代码审查意见
- 触发和查看 CI/CD 结果
- 查看仓库动态（新的 Issue、PR、Release）

### 6.2 团队协作平台

与飞书、Slack 等平台集成。

- 接收来自多个渠道的消息
- 跨平台消息转发和同步
- 在群聊中只在有价值时发言

### 6.3 邮件系统

收发和管理邮件，支撑科研通知、人际沟通、系统告警等场景。

- 收件箱摘要：未读邮件按重要性排序，一句话概括每封
- 草拟回复：根据上下文和用户风格生成回复草稿
- 分类归档：按规则自动归类（工作 / 学术 / 社交 / 通知）
- 定时检查：新邮件到达时通知用户，过滤垃圾和低优先级

### 6.4 日历服务

读写日历事件，支撑日程管理、deadline 跟踪、社交提醒。

- 查询空闲时间段
- 创建 / 修改 / 删除日历事件
- 多日历聚合（工作、个人、学术 deadline）
- 冲突检测：新事件与已有安排冲突时提醒
- 日程摘要："今天有哪些安排""这周有什么 deadline"

### 6.5 搜索引擎

通用互联网搜索，支撑技术调研、学术检索、信息查询等。

- 关键词搜索并返回结构化结果
- 网页内容抓取和摘要
- 支撑 1.4 技术调研中的所有场景
- 可替代人工 Google 的日常信息查询

### 6.6 IoT 中枢

与智能家居平台通信，统一管控家中设备。

- 通过 Home Assistant REST API 查询设备状态
- 调用服务控制设备（开关灯、调温度、开关窗帘等）
- 读取传感器数据（温湿度、光照、用电量）
- 触发自动化场景（回家模式、睡眠模式等）

### 6.7 即时通讯 Bot

通过 Bot 接口主动触达用户或接收指令，不依赖 OpenCami。

- Telegram Bot：最易集成，支持富文本、图片、按钮交互
- 飞书机器人：已有插件基础，适合工作场景
- 接收消息 → 转为任务指令；任务完成 → 推送结果
- 作为 3.3 主动通知的补充渠道

### 6.8 天气与位置服务

查询天气和地理信息，增强日程建议和家居自动化。

- 实时天气和未来预报
- 基于天气的智能建议（出门带伞、开窗通风）
- 地点搜索和路线规划
- 通勤时间估算（辅助日程安排）

### 6.9 云存储同步

与云盘交互，管理跨设备文件。

- 上传 / 下载文件
- 同步科研文献、笔记、项目资料
- 支持主流平台（iCloud、Google Drive、坚果云等）

### 6.10 第三方 API（通用）

按需调用其他外部服务。

- 云服务 API（部署、监控、日志）
- SaaS 工具（设计工具、数据库管理等）
- 以 MCP（Model Context Protocol）插件化方式扩展，不硬编码

## 七、科研技能

### 7.1 文献检索

在学术数据库中搜索论文，替用户完成初步调研。

- 按关键词、作者、年份搜索论文
- 查看引用关系：谁引了这篇、这篇引了谁
- 追踪特定方向的最新进展
- 整理搜索结果为结构化列表（标题、作者、摘要、链接）

### 7.2 论文阅读与摘要

下载论文全文并生成结构化摘要，降低阅读门槛。

- 下载 PDF 并提取全文
- 生成结构化摘要：研究问题 → 方法 → 关键结果 → 局限性
- 横向对比多篇论文的方法和结论
- 回答用户关于论文内容的具体问题

### 7.3 文献管理

管理用户的论文库和引用。

- 维护阅读清单（待读 / 在读 / 已读 + 笔记）
- 导出 BibTeX 引用
- 与 Zotero 同步（可选）
- 按主题、项目、时间组织文献

### 7.4 数据分析与可视化

执行科学计算，生成图表，辅助数据驱动的研究。

- 运行 Python 脚本（pandas / numpy / scipy / matplotlib）
- 统计分析：描述性统计、假设检验、回归
- 生成论文级别的图表
- 分析实验数据，发现趋势和异常

### 7.5 实验管理

记录和对比实验，确保可复现性。

- 记录实验配置（参数、数据集、环境）
- 跟踪实验结果和指标变化
- 对比不同实验的性能差异
- 生成实验报告

### 7.6 学术写作辅助

辅助论文和报告的撰写。

- 根据大纲展开段落
- LaTeX 公式编写和排版
- 语法和表达润色（中英文）
- 格式检查（符合目标期刊/会议要求）

## 八、智能家居技能

### 8.1 设备查询与控制

查看家中设备状态并执行控制操作。

- "客厅灯开着吗" → 查询设备状态并回答
- "把空调调到 26 度" → 调用服务执行
- "关掉所有灯" → 批量操作
- 理解自然语言指令，映射到具体设备和操作

### 8.2 场景管理

创建和触发预设的家居场景。

- 维护场景列表（回家、离家、睡眠、起床、会客等）
- "我要睡了" → 执行睡眠场景（关灯、降温、开加湿器等）
- 新建 / 修改场景：用户口述规则，Agent 转为自动化配置
- 根据时间、传感器数据自动触发

### 8.3 环境监测

持续感知家中环境，异常时主动提醒。

- 温湿度、空气质量、光照等传感器数据汇总
- 趋势分析："最近一周卧室湿度偏低"
- 异常告警：温度骤变、空气质量差、漏水检测
- 建议："室内 CO2 偏高，建议开窗通风"

### 8.4 能耗管理

监控家庭用电，帮助节能。

- 各设备 / 房间用电量统计
- 异常高耗电检测并告警
- 月度能耗报告
- 节能建议（如待机设备关闭）

### 8.5 安防监控

家庭安全相关的监控和告警。

- 门锁状态查询（已锁 / 未锁）
- 异常活动检测（门窗传感器、运动传感器）
- 摄像头画面调取和分析（结合视觉模型）
- 离家模式：自动布防 + 异常时推送通知

## 九、人际管理技能

### 9.1 通讯录与关系图谱

维护一个结构化的联系人知识库。

- 联系人基本信息（姓名、联系方式、所属组织）
- 关系标签（家人、朋友、同事、合作伙伴、同学等）
- 关系备注（认识场景、共同话题、注意事项）
- 互动频率追踪："三个月没和张三联系了"

### 9.2 社交日程

管理与人际关系相关的时间节点。

- 生日、纪念日自动提醒（提前提醒以便准备）
- 定期联络提醒：重要关系设置定期 check-in
- 约会 / 聚会安排：查空闲 → 建议时间 → 创建日程
- 会议准备：约会前提供对方的背景信息和上次互动摘要

### 9.3 沟通辅助

帮助用户更高效、得体地沟通。

- 邮件 / 消息回复草稿：根据上下文和关系生成合适措辞
- 措辞润色：把用户的大白话改为得体的正式表达，或反过来
- 多语言辅助：涉及外语沟通时协助翻译和润色
- 敏感内容提醒：涉及钱、拒绝、投诉等话题时谨慎措辞

### 9.4 人情往来

记录和管理人情账本。

- 送礼 / 收礼记录（时间、对象、物品、金额）
- 随份子记录
- 人情债追踪：谁欠我 / 我欠谁
- 节日送礼建议：根据关系亲密度和对方偏好推荐

### 9.5 会议与协作

辅助正式的人际协作场景。

- 会前：整理议题、准备背景材料
- 会中：记录会议纪要（如有语音输入）
- 会后：整理待办事项、发送跟进邮件
- 跨时区协调：考虑对方时区安排合适的会议时间

### 安装速查

> `setup.sh` 已自动处理以下安装。手动初始化时参考此清单。
> 需要 **Python 3.12+**（`brew install python@3.12`）和 **Docker**（SearXNG 依赖）。

```bash
# ── Homebrew CLI 工具 ──
brew install gh                              # GitHub CLI
brew install steipete/tap/peekaboo           # macOS 桌面 GUI 自动化
brew install himalaya                        # 邮件 CLI (IMAP/SMTP)
brew install rclone                          # 云存储同步
brew install pandoc                          # 文档格式转换

# ── Python 虚拟环境 (.venv, Python 3.12) ──
python3.12 -m venv .venv
source .venv/bin/activate

# STT 代理
pip install -r services/doubao-stt-proxy/requirements.txt

# Web 自动化
pip install playwright && playwright install chromium

# 日历 / 通讯录 / Home Assistant
pip install khal vdirsyncer khard homeassistant-cli

# 科研
pip install paperscout arxivy dblpcli s2cli   # 论文搜索（含依赖）
pip install papis                            # 文献管理
pip install pandas numpy scipy matplotlib    # 数据分析

# ── SearXNG 搜索引擎 (Docker) ──
# start.sh 自动管理，首次运行会拉取镜像并挂载 services/searxng/settings.yml
docker run -d --name searxng --restart unless-stopped \
  -p 8080:8080 \
  -v "$(pwd)/services/searxng/settings.yml:/etc/searxng/settings.yml:ro" \
  searxng/searxng:latest
```
