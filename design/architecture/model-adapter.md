# 模型适配层

模型适配层的目标是保护产品资产，而不是追求兼容所有模型。

## 只负责什么

- 消息格式转换
- 工具 schema 转换
- 流式输出差异适配
- 供应商级认证与参数映射
- 多模态输入的提供商兼容处理

## 明确不负责什么

- 任务生命周期
- 记忆层与知识层语义
- skill 定义与管理
- 审计策略
- 委托边界规则

## 设计原则

1. 适配层要薄，避免再次长成一个“大而全中台”。
2. 只支持真正会长期使用的少数模型家族。
3. 如果切换模型，记忆、知识、任务和工具协议不应推倒重来。
4. 产品应拥有自己的能力资产，而不是寄生在某家原生 memory / skills / native tools 上。
