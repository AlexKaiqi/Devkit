# 项目目标

## 愿景

打造一个 **具备常见通用能力并支持持续扩展专业能力的 AI 管家 / 数字分身**。用户主要通过风铃这个自有移动端客户端交代目标；Telegram 如果保留，默认只作为可选通知或轻量集成工具，而不是需要深度打磨的对等客户端。共享 runtime 负责任务理解、工具调用、异步推进、结果回报与记忆沉淀。

核心理念：
- **分身而非助手** — 有自主决策权，不是被动等指令的工具
- **自有客户端优先** — 随着 LLM 开发成本下降，长期竞争力更可能来自定制软件与专属工作流，而不是寄生在通用 IM 客户端里
- **通用底座优先** — 先把交互、任务、异步、记忆、确认、审计等共性能力做扎实，再扩展专业场景
- **记忆层与知识层是核心难题** — 真正决定产品上限的，不是工具数量，而是能否持续构建、维护并复用对用户有价值的记忆层与知识层资产
- **能力资产自有** — 任务、记忆层、知识层、skill 和工具协议应沉淀在本地 runtime 中，而不是绑定在某个模型供应商的专有能力上
- **可扩展** — 通过 CLI、MCP、spec 驱动文档，让新专业能力能持续增量接入
- **AI 原生** — 项目本身的初始化、维护、扩展都由 AI 驱动
- **全链路本地** — 所有服务本地运行，数据不外泄

## 产品目标

### 1. 建立通用能力底座

优先把所有专业能力都要复用的基础能力做稳：

- 多渠道对话与多模态输入
- 任务生命周期与任务续作
- 工具调用与结果回报
- 定时、并发与主动通知
- 记忆层、知识层、委托边界、审计与安全边界

其中最高优先级难题是：如何让记忆层与知识层既能持续积累，又不会迅速失真、失控或不可维护。

### 2. 建立模型可替换性，保护能力资产

项目需要一层**很薄的模型适配层**，但不追求支持大量模型。

目标是：

- 让记忆层、知识层、任务、skill、工具协议不绑定单一模型
- 把模型差异限制在消息格式、tool schema、prompt 装配等边缘层
- 在需要切换模型或供应商时，不推倒已有能力资产
- 优先支持少数真正会使用的模型家族，而不是做一个"兼容所有模型"的大而全中间层

换句话说，模型是可替换推理内核，产品资产必须掌握在自己的 runtime 里。

### 3. 支持专业能力持续扩展

专业能力不是当前阶段的主矛盾，但架构上必须允许它们低成本接入。

在通用底座之上，优先支持以下高价值方向逐步接入：

- 开发代理
- 知识与科研
- 人际管理
- 智能家居
- 数据与股票分析

新能力应能以较低成本接入，而不是每加一个场景就重写系统。

扩展方式包括：

- 新增独立 spec，定义任务、输入输出和验收标准
- 复用既有通用能力，不重复发明交互/异步/确认机制
- 通过 CLI、MCP、HTTP API 或本地服务接入新工具链

这些专业能力能否真正成立，很大程度上取决于是否拥有可复用的记忆层与知识层资产。

### 4. 保持 AI 原生项目体验

项目本身也应体现 AI 原生范式：

- `git clone` 后 AI 可引导完成初始化
- 文档优先于代码，能力边界优先于实现细节
- 配置、运行、测试、扩展尽量由 AI 协助完成

## 当前迭代重点

1. **打磨通用能力规格** — 任务生命周期与续作、信任模型与委托边界、异步执行、结果回报
2. **优先解决记忆层与知识层维护** — 明确分层、写入策略、来源追踪、去重、修正和复用机制
3. **建立薄模型适配层** — 明确哪些能力属于 runtime，哪些差异留在模型接入边缘层
4. **修复通用底座短板** — SearXNG 网络、任务状态持久化、工具执行可见性
5. **收敛专业能力文档** — 让每个领域通过独立 spec 定义，不再与通用机制混写
6. **补齐可扩展接入路径** — 让邮件、日历、MCP、GUI 自动化等外部能力更容易接入

## 约束条件

- **平台范围**: 先以 macOS (Apple Silicon) + Cursor IDE 为主，不追求一开始跨平台
- **工具选型**: 开源优先、CLI 优先、闭源仅限不可替代
- **Python**: 3.12+，项目内 `.venv` 管理
- **本地优先**: 记忆层、审计和任务状态优先保存在本地可控介质；知识层实现保持解耦
- **模型可替换性**: 不绑定单一模型或供应商；通过薄适配层支持少数可替换模型家族
- **能力资产自有**: 记忆层、知识层、任务、skill 不依赖某家模型的原生 memory / native tools / skills 才能成立
- **渐进扩展**: 不追求一次支持很多模型或很多专业领域，先把底座和少数高价值方向做稳
- **中文优先**: 文档和沟通以中文为主，技术术语保留英文
