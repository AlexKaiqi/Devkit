# 能力需求总览

能力分成两层：

- 通用能力：AI 分身作为产品底座必须具备的长期稳定能力。
- 专业能力：在通用能力之上按领域装配的任务能力包。

本文档只定义“应具备什么能力”，不跟踪当前完成度，也不描述当前代码路径。

| 任务生命周期与续作 | 所有任务都应有状态、回报和续作规则 | [任务生命周期与续作](../core/task-lifecycle.md) |
| 信任模型与委托边界 | 高风险动作必须有明确的确认与边界检查 | [信任模型与委托边界](../core/trust-boundaries.md) |

## 扩展原则

新增一个专业能力前，应先回答四个问题：

1. 它解决的核心用户任务是什么。
2. 它复用了哪些通用能力。
3. 它输出什么结果或附件。
4. 它如何判断自己真的替用户完成了事情。

如果一个新方向只能列工具名单，却没有任务定义、结果形态和验收标准，它还不算专业能力。
