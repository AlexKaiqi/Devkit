# [FUTURE] 知识管理与科研能力

## 背景

用户兼具开发者和研究者身份，需要系统性的知识管理和科研辅助能力。

## 需求

### 知识管理

#### 经验积累

从每次开发中学习，把经验沉淀下来。

- 记录踩过的坑和解决方案
- 记录用户的偏好和习惯
- 记录项目的技术栈、架构特点
- 新经验不覆盖旧经验，形成增量知识库

#### 跨会话记忆

每次对话结束后不失忆，下次能接上。

- 会话开始时自动加载最近的上下文
- 长期重要信息持久保存
- 支持按关键词检索历史记忆

#### 知识检索

在积累的知识中快速找到相关信息。

- 按错误关键词搜索踩坑记录
- 按项目名搜索历史决策
- 按日期范围查看工作日志

### 科研

#### 文献检索

按关键词、作者、年份搜索论文（arXiv / Semantic Scholar / DBLP）。

#### 论文阅读与摘要

下载 PDF 并生成结构化摘要：研究问题 → 方法 → 关键结果 → 局限性。

#### 文献管理

维护阅读清单，导出 BibTeX 引用，按主题组织文献。

#### 数据分析与可视化

运行 Python 脚本（pandas / numpy / scipy / matplotlib），生成论文级别的图表。

#### 实验管理

记录实验配置和结果，确保可复现性。

#### 学术写作辅助

根据大纲展开段落，LaTeX 公式编写，语法润色。

## 验收标准

- [ ] MEMORY.md + memory/ 日志跨会话持久
- [ ] 踩坑记录可按关键词检索
- [ ] paperscout 论文搜索端到端可用
- [ ] PDF → Markdown 转换可用
- [ ] pandas/matplotlib 数据分析可用

## 技术约束

- 记忆层通过 `MEMORY.md + memory/` 承载，知识层能力由独立系统承担
- 科研工具通过 .venv Python 3.12 运行

## 相关决策

- 记忆层用文件存储，知识层与当前项目解耦（2026-03-05）
