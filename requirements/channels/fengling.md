# 语音交互能力

## 背景

用户在碎片化场景（通勤、做饭、散步）需要语音交互。需要低延迟、自然的语音对话体验。

## 需求

### 多渠道对话

通过文字和语音与用户实时沟通，支持多端多渠道。

- **风铃**（Web）：语音+文字双模式，语音问语音答、打字问文字答，大模型音色可选
- **Telegram Bot**：文字 + 语音 + 图片 + 视频，手机随时下指令
- 统一 Agent 后端：风铃与 Telegram 接入同一个 `LocalAgent`
- 回复内容自动分区：对话文本（朗读）+ 附件（代码块，仅展示）
- 输入方式自动匹配回复形态

### 语音输入

- 用户说话 → 豆包 BigModel ASR → 文字 → Agent 执行
- 中文识别准确，延迟低

### 条件语音输出

- 仅语音输入时触发 TTS，文字输入只返回文字
- 大模型音色：豆包语音合成大模型（25+ 音色），风铃右上角可切换
- 流式 TTS：逐句合成，边生成边播放，显著降低首句延迟
- 文本/附件分离：代码和命令不朗读，只展示

### 视觉理解

理解用户发送的图片和视频。

- 用户拍屏幕上的报错截图，Agent 能看懂并分析
- 查看 UI 截图判断样式是否符合预期
- 观看操作录屏，理解用户遇到的问题
- 视频自动抽帧（ffmpeg），兼容所有 OpenAI 兼容 API

### 主动通知

Agent 在关键事件发生时主动找用户。

- 任务完成或失败时立即通知
- 发现异常时告警
- 根据时间和重要性决定是否通知

## 验收标准

- [ ] 语音输入 → 语音回复（逐句流式 TTS）
- [ ] 文字输入 → 纯文字回复（不触发 TTS）
- [ ] 代码块不被朗读
- [ ] 图片/视频可被视觉模型分析
- [ ] 25+ 大模型音色可切换

## 技术约束

- STT: 豆包 BigModel ASR，Whisper API 兼容代理
- TTS: 豆包语音合成大模型 V1 API + `volcano_tts` cluster
- 同一 APPID/TOKEN 同时用于 STT 和 TTS
